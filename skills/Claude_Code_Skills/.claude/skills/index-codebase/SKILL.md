# Purpose

Learn a software project and build a reusable Project Knowledge Base before downstream QA automation.

# When to Use

Use this skill whenever its responsibility applies in the QA pipeline.

# Inputs

Describe all required project artifacts before execution.

# Outputs

Produce structured Markdown suitable for downstream AI agents.

# Responsibilities

- Perform only the responsibilities of this skill.
- Reuse Project Knowledge Base where applicable.
- Never invent missing information.
- State assumptions explicitly.

# Workflow

1. Read inputs.
2. Analyze.
3. Validate completeness.
4. Produce structured output.

# Rules

- Prefer existing project conventions.
- Never modify source code.
- Keep output deterministic.
- Use Markdown headings.

# Handoff

This skill should produce output consumable by the next QA pipeline skill.
